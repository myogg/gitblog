## Project Overview

GitHub Issues-based static blog system. Blog posts are GitHub Issues in `myogg/gitblog`, labels are categories/tags. Python scripts generate a static HTML site deployed to GitHub Pages at `134688.xyz`.

## Data Flow

1. User creates/edits an Issue in `myogg/gitblog` with labels
2. GitHub Actions triggers `generate_readme.yml` (on issue opened/edited) or `deploy.yml` (on labeled)
3. `main.py` fetches issues via PyGithub, generates README.md and backs up issues to `BACKUP/`
4. `tts_generate.py` (optional) generates Chinese TTS audio via read-aloud API, stores in R2 or `static/tts/`
5. `generate_page.py` fetches all issues, renders Jinja2 templates, outputs static HTML files
6. Site is deployed to GitHub Pages

## Key Scripts

- `generate_page.py` — Main static site generator. Fetches GitHub Issues, renders all HTML pages, search index, sitemap, tag pages, RSS feed. Caches GitHub API responses for 6 hours in `github_cache.json`
- `main.py` — Lightweight script for README.md regeneration and issue backup (used by `generate_readme.yml` workflow)
- `tts_generate.py` — TTS audio generation via read-aloud API. Supports R2 (production) and `--local` (dev, files go to `static/tts/`) modes. Outputs `tts_cache.json` consumed by `generate_page.py`

## Templates (`templates/`)

- `base.html` — Main layout with custom Puma theme header (banner image, search bar, nav, theme toggle)
- `blog.html` — Homepage with article list grouped by year, pagination support
- `article.html` — Article page with reading progress bar, Waline comments, TTS audio player, related articles, markdown rendering via `github-markdown-css`
- `tag.html`, `search.html`, `about.html` — Tag listing, client-side search, about page

## Static Assets (`static/`)

- `style.css` — Full custom CSS with CSS variables, dark mode support, responsive breakpoints
- `fonts.css` — Local Inter (variable, weights 300–600) and JetBrains Mono (weights 400–500) via `@font-face`
- `script.js` — Theme toggle, header scroll effects, back-to-top button, search functionality
- Noto Sans SC loaded from Google Fonts for Chinese text
- Cache busting uses `?v=N` query param — increment when deploying CSS/JS changes

## Generated Files (never edit manually)

- `index.html` — Homepage, generated by `generate_page.py`
- `articles/` — Per-article HTML pages
- `search.html`, `about.html`, `tag/` pages — Generated by `generate_page.py`
- `feed.xml` — RSS feed
- `sitemap.xml`, `robots.txt` — SEO files
- `static/search-index.json` — Client-side search index

## Key Conventions

- Article summaries use `<!-- more -->` separator in issue body (optional)
- Article tags can be embedded via YAML frontmatter or `tags:` line in issue body
- TTS audio URLs stored in `tts_cache.json`, loaded by `generate_page.py`
- GitHub API responses cached 6 hours in `github_cache.json` — delete to force refresh
- Only repo owner's issue events trigger deployment; other users' events are filtered

## Common Commands

```bash
pip install -r requirements.txt

# Set your GitHub token
export G_TT=your_github_token

# Full site generation
python generate_page.py

# Single article
python generate_page.py --issue 17

# TTS (local mode, no R2 needed)
python tts_generate.py --local
python tts_generate.py --issue 30

# README + backup
python main.py $G_TT myogg/gitblog [--issue_number N]

# Local preview
python -m http.server 8000

# Clear API cache
rm github_cache.json
```

## Environment Variables

- `G_TT` — GitHub personal access token (required)
- `TTS_API_URL`, `TTS_API_TOKEN` — TTS API endpoint
- `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET`, `R2_PUBLIC_URL` — R2 storage for TTS (optional, omit for `--local`)

## CI/CD

- **generate_readme.yml**: Triggered by issue opened/edited, issue comment, or push to main affecting `main.py`. Runs `main.py` to update README.md and commits issue backups.
- **deploy.yml**: Triggered manually or when an issue is labeled by repo owner. Runs `tts_generate.py` → `generate_page.py`, commits TTS cache, deploys to GitHub Pages. Has concurrency guard per issue number.
